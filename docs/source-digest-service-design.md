# AI Outlook (PDF) — 서비스 설계 스펙

> 작성 2026-06-11. aitimes.kr 콘텐츠 역추적 기반. 기존 ainews-brief 하네스의 **자매 서브 하네스**.

## 0. 한 줄 정의
aitimes.kr이 한글화하는 **상류(원천) 1차 소스를 직접 감시**해, 그날 **가장 핫한 이슈 1건을 선별**하고
원천 원문 + 관련 소스를 끌어와 **심층 요약·분석**한 **A4 1~2페이지 PDF 분석 브리프**를 만들어
**웹포털에 게시(필요 시 PDF 다운로드)** 한다. 독자는 **과학기술 연구자·정책자**.

> **설계 전환(2026-06-12):** "핵심 3~4건 + 단신 얕은 요약" → **"가장 핫한 1건 심층 분석"**.
> 연구자·정책자에게는 다건 나열보다 한 이슈를 깊이 파고든 분석 브리프가 가치가 크다는 사용자 판단 반영.

## 1. aitimes.kr 역추적 결론
aitimes.kr = 원천 1차 소스를 **수시간~하루 늦게 한글 번역·재구성하는 애그리게이터**.
출처 링크는 거의 없으나 본문 단서로 원천이 드러남. 검증된 원천 패턴:
- AWS 그라비톤5 → AWS 뉴스룸/re:Invent
- 보쉬 로보틱스 → Bosch 보도자료/BCW 2026 컨퍼런스
- 클로드·젬마 출시 → Anthropic/Google 공식 블로그
- 트럼프 AI 행정명령 → White House 1차 발표
- 아모데이 발언 → 외신/인터뷰
→ 상류 5계층(기업공식·컨퍼런스·정부·외신·논문)을 직접 감시하면 구조적으로 선행 가능.

## 4. 파이프라인 (핫 이슈 1건 심층)
```
①원천 스캔 → ②핫이슈 랭킹·1건 선별 → ③심층 수집 → ④OpenAI 본문서술 → ⑤Claude 검증·조립 → ⑥PDF·포털 게시
RSS/블로그 폴링   교차 가중 점수화          선택건 원문+        개요·주요내용·       사실성 대조·     A4 1~2p
신규 검출(state)  Top1 + 차순위 후보로그     관련 원천 다수fetch  시사점(gpt-4o)      환각 교정·HTML   +PDF
```
- 신규 검출: 소스별 마지막 처리 항목 ID/URL/타임스탬프를 `state.json`에 저장 → 중복 방지.
- **역할 분담:** 수집·교차검증(1차 출처)은 Claude collector, 본문 서술은 OpenAI(`scripts/summarize_openai.py`, gpt-4o), 사실성 검증·교정·조립은 Claude analyst. 키 부재 시 analyst 직접 작성으로 폴백.

### 4-1. 핫이슈 랭킹 (②) — "가장 핫한 1건" 선별 로직
스캔된 신규 항목을 다음 가중치로 점수화해 **1위 1건**만 선정(2~3위는 후보로 로그):
- **교차 노출도** — 여러 원천(기업+외신+논문 등)에서 동시에 다뤄질수록 ↑
- **파급도** — 모델/제품 출시·정책 변화·대형 투자/M&A·반도체/인프라 등 구조적 사건
- **신규성** — 기존 흐름 대비 새로운 정도
- **연구·정책 적합성** — 과학기술 연구자/정책자 관심도(벤치마크·방법론·규제·표준·안전)
- 동점 시 **원천 1차성**(공식 발표·논문 > 2차 보도)·최신성 우선.

### 4-2. 심층 수집(③) → 본문 서술(④) → 검증(⑤)
선택된 1건에 대해 **원천 원문 + 관련 소스 다수**(경쟁사 반응·해설 외신·관련 논문·정부 입장)를 Claude collector가 수집·교차검증해 `03_dossier.json`을 만든다(deep-research식 팬아웃 검색→fetch→교차검증→인용).
그 검증된 dossier를 근거로 **OpenAI(`scripts/summarize_openai.py`, gpt-4o)가 개요·주요내용·시사점을 한글 서술**(`04_analysis.json`)하고, **Claude analyst가 dossier와 대조해 환각·과장을 교정**한다. 출처·단신 링크는 스크립트가 dossier에서 그대로 옮겨 정확성을 보장(LLM 미경유). 매 회차 단일 이슈 한정이라 비용·시간 통제 가능, 1~2p 분량에 맞춰 캘리브레이션.

## 5. PDF 레이아웃 (A4 1~2p) — 심층 분석 브리프
본문 서술은 OpenAI(gpt-4o)가 dossier 근거로 생성, Claude analyst가 검증·교정.
- **헤더:** 제목 · 날짜·회차(am/pm) · `오늘의 핵심 이슈` · [주요 키워드 배지 5개 이내]
- **Keynote (박스, 3~4개):** 무슨 일인지 + 왜 중요한지 핵심 요지
- **① 개요:** 무슨 일이 일어났고 왜 중요한지 한 문단 도입부(큰 그림)
- **② 주요내용 (심층, 가장 두껍게):** 사실관계 + 기술·수치·벤치마크를 연구자 수준 디테일로(주체·시점·메커니즘·사양·모델명 원문 병기)
- **③ 시사점 (주제 적응형 관점):** 이슈 성격에 따라 관점을 **택일·가중**하고 전망·한계까지 녹인다 — 무엇이 어떻게 바뀌는지.
  - 모델/기술/논문 등 → 🔬 **연구·개발 관점**(방법론·재현성·후속연구·한계) 중심
  - 규제/표준/거버넌스/투자 등 → 🏛 **정책 관점**(규제·표준·산업·안보·노동) 중심
  - 경계 사안은 두 관점 병기하되 주(主) 관점을 명시
- **④ 출처·더 읽을거리:** 1차 원천 + 관련 원천 직링크 목록
- **그 외 주목 단신(푸터):** 2~3건 제목+링크만으로 breadth 보강 — **본문 1건 깊이는 그대로 유지**
- A4 / 여백 15mm / 본문 9~10pt / Noto Sans KR 임베드 / 핵심 도표·이미지 로컬 임베드
- **분량: 무조건 A4 1~2p 고정.** 내용이 넘치면 본문 압축·단신 축소로 맞춤(상한 초과 불가).

## 6. 구현 구성요소 (예정)
- 신규 에이전트: `source-digest-collector` (기업/컨퍼런스 원천 폴링·신규검출·본문/이미지 수집)
- 신규 스킬: `source-pdf-digest` (선별·요약·PDF 조립 절차) + `references/sources.yaml`
  - `scripts/scan_sources.py` (RSS/블로그 폴링·state 갱신)
  - `scripts/html_to_pdf.py` (WeasyPrint 렌더)
- 기존 재사용: global-news-collector, news-collector(arXiv/HF), policy-collector
- 오케스트레이터: 신규 또는 ainews-orchestrator 확장 (1일 2회 분기)
- 포털: `webapp/` 확장 — 리포트 목록 + PDF 다운로드 라우트
- 예약: 1일 2회 cron (아침 06:00 / 저녁 시각 KST)

## 7. 리스크/대응
- RSS 우선이라 스크레이핑 의존도 낮음. RSS 없는 곳만 폴링·파서 실패 시 1회 재시도·로그.
- 한글 폰트 깨짐 → Noto Sans KR @font-face 임베드.
- 이미지 핫링크 → 로컬 다운로드 후 임베드.
- 저작권 → 요약 + 원문 직링크만, 전문 복제 금지, 출처 명시.
